# v1.1.0 - Best Practice Template

## Why

Refactor telegram component to follow zylos component best practices, making it the reference template for future component development.

## What

### Structure Changes
- `send.js` moved from `src/` to root directory (comm-bridge interface standard)
- Added `CHANGELOG.md` for version tracking (zylos-pm requirement)
- Added `LICENSE` file (MIT)
- Added `upgrades/` directory for version documentation

### SKILL.md Updates
- Added `type: communication` (required field)
- Added `dependencies: [comm-bridge]`
- Fixed documentation paths

### Code Changes
- `install.js`: use `--omit=dev` instead of `--production`
- `upgrade.js`: use `--omit=dev` instead of `--production`

## How

### Upgrade from 1.0.x
```bash
zylos-pm upgrade telegram
```

The upgrade is automatic. Key changes:
1. `send.js` is now at root - update any direct references
2. Use `node ~/.claude/skills/telegram/send.js` (not `src/send.js`)

### Manual Upgrade
```bash
cd ~/.claude/skills/telegram
git pull origin main
npm install --omit=dev
pm2 restart zylos-telegram
```

### Rollback
```bash
cd ~/.claude/skills/telegram
git checkout v1.0.4
pm2 restart zylos-telegram
```
